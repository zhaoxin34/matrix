# 修复问题的步骤

## step1 问题重现和分析

根据用户提出问题重现问题，以便确认用户说的问题被你认同，一般重现问题的方法可以是使用playwright-cli skills访问页面，使用curl 访问页面，或者查看后端日志，如果上述方法都失效了，那么读取可能发生错误的代码，来猜测问题

## step 修改问题

依据问题的原因思考解决问题方法，但一定要遵守开发规范

## 验证问题是否被修复

根据重现问题的方式进行验证，比如重现问题的使用的是curl，那么验证问题是否被修复，最好还是用curl。

## 检查代码质量

使用/simplify 简化代码

使用make line && make format && make type-check 验证代码质量
