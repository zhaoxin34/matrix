## 新增需求

### 需求：创建员工
系统**必须**允许授权用户创建员工档案，包含工号（唯一）、姓名、手机号、邮箱、岗位、主属部门 ID 和入职日期。员工档案与系统账号（User）通过 `user_employee_mapping` 关联表解耦，创建员工时可选择绑定一个尚未关联员工的系统账号，也可不绑定（如待入职状态）。

#### 场景：成功创建员工（不绑定账号）
- **当** 管理员提交包含所有必填字段且工号唯一的创建请求，未指定 user_id 时
- **则** 系统创建员工记录并返回新员工的数据，不创建 user_employee_mapping 记录

#### 场景：成功创建员工并绑定账号
- **当** 管理员提交创建请求，并指定一个尚未绑定任何员工的 user_id 时
- **则** 系统创建员工记录，并在 user_employee_mapping 中创建关联记录

#### 场景：工号重复时拒绝创建
- **当** 管理员提交的创建请求中包含已存在的工号时
- **则** 系统返回 400 错误，提示工号必须唯一

#### 场景：绑定已关联员工的账号时拒绝
- **当** 管理员指定的 user_id 已在 user_employee_mapping 中存在记录时
- **则** 系统返回 400 错误，提示该账号已绑定其他员工

### 需求：绑定或解绑员工与系统账号
系统**必须**允许授权用户为已存在的员工档案绑定或解绑系统账号，且保证 One-to-Zero-or-One 约束。

#### 场景：为员工绑定账号
- **当** 管理员为一名未绑定账号的员工指定一个未被占用的 user_id 时
- **则** 系统在 user_employee_mapping 中创建关联记录

#### 场景：解绑员工账号
- **当** 管理员请求解除某员工与账号的绑定时
- **则** 系统删除对应的 user_employee_mapping 记录，员工档案保留

### 需求：编辑员工
系统**必须**允许授权用户更新现有员工档案的任意字段。

#### 场景：成功修改员工岗位
- **当** 管理员对已存在的员工提交合法的更新请求时
- **则** 系统持久化变更并返回更新后的员工数据

#### 场景：编辑不存在的员工
- **当** 管理员对不存在的员工 ID 提交更新请求时
- **则** 系统返回 404 错误

### 需求：删除员工
系统**必须**允许授权用户对员工记录执行软删除，将其状态设为 offboarding 并记录离职日期。

#### 场景：离职员工
- **当** 管理员请求删除一名已存在的员工时
- **则** 系统将该员工状态设为 offboarding，并将 dimission_date 记录为当天日期

### 需求：为员工分配辅属部门
系统**必须**允许员工在主属部门之外关联一个或多个辅属组织单元。

#### 场景：添加辅属部门
- **当** 管理员为员工添加辅属部门时
- **则** 系统创建一条关联该员工与该部门的 employee_secondary_unit 记录

#### 场景：移除辅属部门
- **当** 管理员移除员工的某个辅属部门时
- **则** 系统删除对应的 employee_secondary_unit 记录

### 需求：列表查询与筛选员工
系统**必须**提供员工列表接口，支持按组织单元（含后代）筛选、按状态筛选，以及按关键词（姓名、工号）搜索。

#### 场景：按部门筛选员工
- **当** 用户向 GET /api/v1/employees 发送带有 unit_id 筛选参数的请求时
- **则** 系统在用户权限范围内，返回所有主属部门或辅属部门属于该单元及其后代的员工

#### 场景：按姓名搜索员工
- **当** 用户向 GET /api/v1/employees 发送带有关键词查询参数的请求时
- **则** 系统返回姓名或工号包含该关键词的员工列表

### 需求：批量导入员工（Excel）
系统**必须**允许授权用户通过上传 Excel 文件批量创建员工记录。

#### 场景：成功批量导入
- **当** 管理员上传一个格式合法的员工数据 Excel 文件时
- **则** 系统逐行验证，为合法行创建员工记录，并返回包含成功数量和行级错误信息的汇总报告

#### 场景：含错误行的部分导入
- **当** 管理员上传的 Excel 文件中部分行存在验证错误（例如工号重复）时
- **则** 系统导入合法行，跳过错误行，并返回每一错误行的详细错误信息

### 需求：导出员工列表（Excel）
系统**必须**允许授权用户将当前筛选结果的员工列表导出为 Excel 文件。

#### 场景：导出员工列表
- **当** 用户请求 GET /api/v1/employees/export（可附带筛选参数）时
- **则** 系统在用户权限范围内，返回包含所有符合条件员工的 Excel 文件
