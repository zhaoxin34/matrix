# 快消品电商平台 用户端 PRD

**版本：** V1.0
**日期：** 2026-04-15
**状态：** 初稿
**负责人：** 产品团队

---

## 1. 执行摘要

我们正在为快消品（FMCG）电商平台构建完整的用户端产品体系，目标是为消费者提供便捷、高效的在线购物体验。快消品具有高复购率、低客单价、强调配送时效的特点，用户端产品设计将围绕"快速找到商品"和"顺畅完成购买"两个核心场景展开。

本PRD聚焦于用户端功能，涵盖商品浏览、搜索、购物车、结算支付、会员体系、评价系统、客服售后等核心模块。通过清晰的购物流程设计和完善的用户中心，提升转化率、复购率和用户满意度。

**预期成果：** 构建完整的用户端产品功能框架，为首期MVP版本提供可落地的产品需求依据。

## 2. 问题陈述

### 2.1 市场背景

快消品电商行业呈现以下特征：

| 特征 | 说明 |
|------|------|
| **高复购率** | 食品饮料、日化用品消费周期短，用户需频繁补货 |
| **低客单价** | 单次购买金额较低（通常50-200元） |
| **注重时效** | 用户期望快速收货，不愿等待 |
| **价格敏感** | 促销、优惠券对购买决策影响大 |
| **决策简单** | 商品标准化程度高，购买决策快 |

### 2.2 用户核心痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| **找商品难** | SKU众多，分类复杂，难以快速找到目标商品 | 浏览放弃率高 |
| **比价困难** | 同类商品价格难以比较 | 决策效率低 |
| **购物车放弃** | 结算流程繁琐、运费不透明 | 转化率低 |
| **支付不安全** | 对支付安全性存疑 | 支付失败率高 |
| **售后困难** | 退换货流程不清晰 | 满意度低 |
| **缺乏引导** | 新用户不知道有哪些优惠 | 首单转化低 |

### 2.3 行业对标

| 平台 | 优势 | 可借鉴点 |
|------|------|----------|
| **天猫超市** | 商品丰富、次日达 | 品类管理、促销活动 |
| **京东超市** | 物流快速、品质保障 | 配送时效展示 |
| **拼多多** | 价格低、社交裂变 | 拼团、分享优惠 |
| **美团闪购** | 即时配送、30分钟达 | 快速履约体验 |

## 4. 战略背景

### 4.1 产品定位

| 维度 | 定位 |
|------|------|
| **平台类型** | 垂直快消品B2C电商平台 |
| **目标市场** | 一二线城市，追求品质生活的年轻家庭及上班族 |
| **竞争策略** |精选SKU + 优质物流 + 会员价值 |
| **差异化** | 极简购物流程、智能化推荐、会员专属优惠 |

### 4.2 产品分阶段规划

| 阶段 | 目标 | 周期 | 核心功能 |
|------|------|------|----------|
| **MVP** | 验证核心购物流程 | 0-3个月 | 商品浏览、购物车、结算支付、订单管理、基础会员 |
| **V1.0** | 完善用户体系 | 3-6个月 | 会员等级、积分体系、评价系统、促销工具 |
| **V1.5** | 增强社交与营销 | 6-9个月 | 分享有礼、拼团、优惠券中心、签到体系 |
| **V2.0** | 智能化升级 | 9-12个月 | 智能推荐、个性化首页、语音搜索 |

### 4.3 本PRD范围

本PRD聚焦于**MVP阶段用户端功能**，包含：

- 商品浏览与展示
- 搜索与筛选
- 购物车
- 结算与支付
- 订单管理
- 用户中心基础功能

**营销功能**（促销、优惠券、社交分享）将在V1.0版本规划，本PRD暂不展开。

### 5.2 核心用户流程

#### 5.2.1 浏览-购买流程

```
用户打开APP
    ↓
首页浏览 / 搜索商品 / 分类查找
    ↓
进入商品详情页
    ↓
选择SKU规格 → 加入购物车 / 立即购买
    ↓
购物车确认 → 修改数量 / 删除商品 / 使用优惠
    ↓
去结算
    ↓
选择收货地址 → 选择支付方式 → 提交订单
    ↓
支付成功 → 订单完成
    ↓
查看订单状态 → 确认收货
    ↓
评价商品
```

#### 5.2.2 用户中心流程

```
进入用户中心
    ↓
账户管理：查看/修改个人信息
    ↓
收货地址：新增/编辑/删除地址
    ↓
我的订单：查看全部/进行中/已完成订单
    ↓
优惠券：查看可用/已使用/已过期
    ↓
消息中心：查看系统通知/订单通知
    ↓
设置：退出登录/清除缓存
```

## 7. 用户故事与需求

### 7.1 Epic：商品浏览与发现

**Epic假设：** 我们相信提供一个清晰的分类体系和强大的搜索功能，能让用户快速找到目标商品，从而提升商品详情页访问量和加购率。

#### 用户故事 1.1：多级分类浏览

**角色：** 作为用户，我希望通过分类导航快速找到商品大类，再逐级细化到具体品类。

**验收标准：**

- [ ] 首页展示一级分类入口，分类图标支持横向滑动
- [ ] 点击分类进入二级分类页，展示该大类下的所有中类
- [ ] 二级分类页顶部显示当前路径（如：首页 > 饮料 > 碳酸饮料）
- [ ] 点击中类进入商品列表页，展示该小类的所有商品
- [ ] 分类数据从后端接口获取，支持运营配置

#### 用户故事 1.2：商品筛选与排序

**角色：** 作为用户，我希望对商品列表进行筛选和排序，快速找到最合适的商品。

**验收标准：**

- [ ] 商品列表页顶部展示"筛选"和"排序"入口
- [ ] 筛选支持：品牌、价格区间、规格/包装、库存状态
- [ ] 筛选条件可多选，支持重置筛选
- [ ] 排序支持：销量优先、价格升序、价格降序、新品优先、评分优先
- [ ] 筛选/排序后列表即时更新，不刷新页面

#### 用户故事 1.3：智能搜索

**角色：** 作为用户，我希望通过搜索快速找到目标商品，并且搜索能智能联想。

**验收标准：**

- [ ] 搜索栏位于页面顶部，始终可见
- [ ] 输入关键词时，下方展示联想词列表（最多10个）
- [ ] 联想词包含热搜词、历史搜索词、商品名称
- [ ] 点击搜索后进入搜索结果页
- [ ] 搜索结果页支持在结果内继续筛选和排序
- [ ] 搜索无结果时展示"未找到相关商品"提示，并推荐热门商品

#### 用户故事 1.4：商品详情页浏览

**角色：** 作为用户，我希望在商品详情页看到商品的完整信息，包括图片、价格、规格、库存、详情描述。

**验收标准：**

- [ ] 商品轮播图支持左右滑动，支持点击放大
- [ ] 页面展示：商品名称、价格（原价+促销价）、促销标签
- [ ] SKU选择器：如有多种规格（如口味、容量），需用户选择后显示对应价格和库存
- [ ] 库存状态实时显示：充足/"仅剩X件"/"缺货"
- [ ] 商品详情区展示：规格参数、食用/使用方法、成分配料、生产日期、保质期
- [ ] 页面底部展示评价入口，显示平均评分和评价数量
- [ ] 页面底部展示"相似商品推荐"

### 7.2 Epic：购物车与结算

**Epic假设：** 我们相信优化购物车到支付的完整流程，减少不必要的信息填写和页面跳转，能显著提升支付转化率。

#### 用户故事 2.1：购物车管理

**角色：** 作为用户，我希望在购物车中方便地管理待购买的商品，包括修改数量、删除商品、查看优惠提示。

**验收标准：**

- [ ] 购物车页面展示商品列表：商品图片、名称、规格、单价、数量、小计
- [ ] 支持修改商品数量：+/-按钮，最小为1，最大为库存数
- [ ] 支持删除商品，删除前二次确认
- [ ] 购物车顶部显示已享受的优惠（如满减已满足）
- [ ] 如有商品无货或价格变化，需明显提示用户
- [ ] 购物车底部显示商品总金额和运费（满足免运条件时显示"免运费"）
- [ ] 底部固定"去结算"按钮

#### 用户故事 2.2：收货地址管理

**角色：** 作为用户，我希望在结算时方便地选择和管理收货地址。

**验收标准：**

- [ ] 地址列表展示所有收货地址，显示默认地址标签
- [ ] 支持新增收货地址，填写：收货人姓名、手机号、省市区、详细地址、邮编
- [ ] 支持编辑已有地址
- [ ] 支持删除地址（默认地址不可删除，需先设置其他地址为默认）
- [ ] 支持设置默认地址
- [ ] 地址信息需进行合法性校验（手机号格式、省市区必填等）

#### 用户故事 2.3：订单确认与提交

**角色：** 作为用户，我希望在订单确认页清晰看到订单的所有信息，确认无误后提交订单。

**验收标准：**

- [ ] 订单确认页展示：商品列表、收货地址、支付方式、订单金额明细
- [ ] 金额明细包含：商品总价、运费（若未满免运门槛）、优惠抵扣（若有）、应付总额
- [ ] 支持选择支付方式：微信支付、支付宝（默认微信支付）
- [ ] 支持选择配送时间（仅支持精确到日期，不支持时段）
- [ ] 支持添加订单备注（选填，最多50字）
- [ ] 提交订单前需确认用户已阅读并同意《用户协议》《隐私政策》
- [ ] 点击"提交订单"后进入支付流程

#### 用户故事 2.4：支付流程

**角色：** 作为用户，我希望通过微信或支付宝安全便捷地完成支付。

**验收标准：**

- [ ] 订单提交后跳转到支付引导页
- [ ] 页面展示订单号、应付金额、支付二维码（微信/支付宝）
- [ ] 用户扫码后实时查询支付状态
- [ ] 支付成功后跳转到支付成功页，显示订单号
- [ ] 支付失败时展示失败原因，提供"重新支付"入口
- [ ] 支付超时（如30分钟未支付）自动取消订单，并提示用户
- [ ] 支付过程需记录日志，便于排查问题

### 7.3 Epic：订单与售后

**Epic假设：** 我们相信提供清晰的订单管理和便捷的售后入口，能提升用户满意度和复购率。

#### 用户故事 3.1：订单列表与详情

**角色：** 作为用户，我希望方便地查看我的所有订单，了解订单状态和物流信息。

**验收标准：**

- [ ] 订单中心包含Tab：全部/待付款/待发货/待收货/已完成/已取消
- [ ] 订单列表展示：订单号、下单时间、商品缩略信息、订单状态、订单金额
- [ ] 点击订单进入订单详情页
- [ ] 订单详情页展示：订单状态、完整商品列表、收货地址、支付信息、物流信息（若已发货）
- [ ] 物流信息实时更新，展示物流轨迹

#### 用户故事 3.2：订单操作

**角色：** 作为用户，我希望对不同状态的订单进行相应操作。

**验收标准：**

- [ ] 待付款订单：显示"取消订单"和"去支付"按钮
- [ ] 取消订单需选择取消原因（二次确认）
- [ ] 待发货订单：显示"催发货"按钮（按钮点击后需有限流控制）
- [ ] 待收货订单：显示"确认收货"按钮，显示物流信息入口
- [ ] 确认收货前需确认已收到商品
- [ ] 已完成订单：显示"申请售后"和"再次购买"按钮
- [ ] 已取消订单：显示"删除订单"和"再次购买"按钮

#### 用户故事 3.3：售后申请

**角色：** 作为用户，我希望在商品有问题时方便地申请售后。

**验收标准：**

- [ ] 仅已完成订单可申请售后
- [ ] 售后类型：仅退款、退货退款
- [ ] 申请时需选择/填写：售后原因（必填）、退款金额（仅退款时填写）、问题描述、图片上传（选填，最多3张）
- [ ] 提交后生成售后工单，进入售后处理流程
- [ ] 用户可在订单详情页查看售后进度和结果
- [ ] 退款原路返回，审核通过后2-7个工作日到账

---

### 7.4 Epic：用户中心

**Epic假设：** 我们相信通过完善的用户中心功能，帮助用户管理账户、地址、资产，能提升用户粘性和数据完整性。

#### 用户故事 4.1：账户信息管理

**角色：** 作为用户，我希望在用户中心管理我的账户信息。

**验收标准：**

- [ ] 用户中心首页展示：用户头像、昵称、会员等级
- [ ] 支持修改昵称（输入框，限20字）
- [ ] 支持修改头像（选择图片或拍照）
- [ ] 支持绑定/更换手机号（需验证原手机号或短信验证码）
- [ ] 支持修改登录密码（需验证原密码）
- [ ] 支持设置支付密码（6位数字）

#### 用户故事 4.2：收货地址管理

**角色：** 作为用户，我希望方便地管理我的收货地址。

**验收标准：**

- [ ] 地址列表展示所有地址，默认地址置顶显示
- [ ] 支持新增地址：收货人、手机号、省市区、详细地址
- [ ] 地址省市区支持联动选择
- [ ] 支持编辑已有地址
- [ ] 支持删除地址
- [ ] 支持设置默认地址
- [ ] 最多保存20个收货地址

#### 用户故事 4.3：消息中心

**角色：** 作为用户，我希望在消息中心查看平台发送的各种通知。

**验收标准：**

- [ ] 消息中心包含Tab：全部/订单消息/系统通知
- [ ] 订单消息类型：订单状态变更、发货提醒、物流更新、退款通知
- [ ] 系统通知类型：活动公告、系统维护、积分变动
- [ ] 每条消息显示：图标、标题、时间、简要内容
- [ ] 点击消息进入详情页
- [ ] 未读消息显示红点数字
- [ ] 支持一键清空已读

### 7.5 Epic：评价系统

**Epic假设：** 我们相信通过完善的评价体系，帮助其他用户做出购买决策，同时为商家提供反馈。

#### 用户故事 5.1：商品评价

**角色：** 作为已购买并收货的用户，我希望对购买的商品进行评价，帮助其他用户了解商品。

**验收标准：**

- [ ] 确认收货后展示"去评价"入口
- [ ] 评价表单：选择评分（1-5星）、填写评价内容（选填，最多500字）、上传图片（选填，最多6张）
- [ ] 提交评价后显示"评价成功"
- [ ] 评价后用户可进行追加评价（收货后30天内）
- [ ] 同一商品仅可评价一次

#### 用户故事 5.2：查看评价

**角色：** 作为准备购买的用户，我希望查看其他用户对该商品的评价，作为购买参考。

**验收标准：**

- [ ] 商品详情页底部展示评价入口
- [ ] 评价列表页展示：用户头像、昵称、评分、评价内容、评价图片、评价时间
- [ ] 支持按"全部/有图/好评/差评"筛选评价
- [ ] 支持按时间排序（默认）或按热度排序
- [ ] 显示评价总数和平均评分
