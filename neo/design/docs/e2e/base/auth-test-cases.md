## 1. 功能测试

### TC-F-001: 手机号注册成功

- **需求**: 用户能够通过手机号、验证码、密码完成注册
- **优先级**: High
- **前置条件**:
  - 手机号未注册
  - 系统可发送验证码
- **测试步骤**:
  1. 访问注册页面 `/register`
  2. 输入用户名（可选）：`测试用户`
  3. 输入手机号：`13800138001`
  4. 点击「获取验证码」按钮
  5. 输入收到的 6 位验证码
  6. 输入密码：`Abcd1234`
  7. 确认密码：`Abcd1234`
  8. 勾选「我已阅读并同意服务条款和隐私政策」
  9. 点击「注册」按钮
- **预期结果**:
  - 注册成功
  - 自动登录并跳转至首页
  - 返回 user_id 和 token
- **后置条件**: 用户账号已创建，可使用该手机号登录

### TC-F-002: 密码登录成功

- **需求**: 用户能够通过手机号+密码登录
- **优先级**: High
- **前置条件**:
  - 用户已注册且账号状态为 active
  - 知晓正确密码
- **测试步骤**:
  1. 访问登录页面 `/login`
  2. 确认已选择「密码登录」选项卡
  3. 输入手机号：`13800138001`
  4. 输入密码：`Abcd1234`
  5. （可选）勾选「记住我」
  6. 点击「登录」按钮
- **预期结果**:
  - 登录成功
  - 跳转至首页
  - 返回 user_id 和 token
  - Token 存储在 httpOnly Cookie 中

### TC-F-003: 验证码登录成功

- **优先级**: High

  5. 输入收到的 6 位验证码
  6. 点击「登录」按钮
- **预期结果**:

### TC-F-004: 发送验证码

## 2. 边界测试

### TC-E-001: 注册时手机号已存在

- **需求**: 手机号唯一性校验
- **优先级**: High
- **前置条件**: 手机号 `13800138001` 已注册
- **测试步骤**:
  1. 访问注册页面 `/register`
  2. 输入手机号：`13800138001`
  3. 输入密码：`Abcd1234`
  4. 确认密码：`Abcd1234`
  5. 勾选协议
  6. 点击「注册」按钮
- **预期结果**:
  - 返回错误码 `2002`（User Already Exists）
  - 提示「该手机号已注册」

### TC-E-002: 注册时密码不符合规范

- **需求**: 密码需 8-20 位，包含字母和数字
- **优先级**: High
- **前置条件**: 手机号未注册
- **测试步骤**:
  1. 访问注册页面 `/register`
  2. 输入手机号：`13800138999`
  3. 输入密码（仅数字）：`12345678`
  4. 确认密码：`12345678`
  5. 勾选协议
  6. 点击「注册」按钮
- **预期结果**:
  - 返回错误码 `1001`（Invalid Parameter）
  - 提示密码需包含字母和数字

### TC-E-003: 注册时两次密码不一致

- **需求**: 确认密码必须与密码一致
- **优先级**: High
- **前置条件**: 手机号未注册
- **测试步骤**:
  1. 访问注册页面 `/register`
  2. 输入手机号：`13800138999`
  3. 输入密码：`Abcd1234`
  4. 确认密码：`Abcd1235`
  5. 勾选协议
  6. 点击「注册」按钮
- **预期结果**:
  - 提示两次密码不一致
  - 无法提交

### TC-E-004: 登录时密码错误

- **需求**: 密码错误时给出提示
- **优先级**: High
- **前置条件**: 用户已注册，正确密码为 `Abcd1234`
- **测试步骤**:
  1. 访问登录页面 `/login`
  2. 输入手机号：`13800138001`
  3. 输入错误密码：`Abcd1235`
  4. 点击「登录」按钮
- **预期结果**:
  - 返回错误码 `1002`（Invalid Password）
  - 提示「密码错误」

### TC-E-005: 登录时用户不存在

- **需求**: 用户不存在时给出提示
- **优先级**: High
- **前置条件**: 手机号未注册
- **测试步骤**:
  1. 访问登录页面 `/login`
  2. 输入手机号：`13900000000`
  3. 输入密码：`Abcd1234`
  4. 点击「登录」按钮
- **预期结果**:
  - 返回错误码 `2001`（User Not Found）
  - 提示「用户不存在」

### TC-E-006: 登录时用户已被禁用

- **需求**: 禁用用户无法登录
- **优先级**: High
- **前置条件**: 用户已注册但被禁用（is_active=false）
- **测试步骤**:
  1. 访问登录页面 `/login`
  2. 输入被禁用用户的手机号
  3. 输入正确密码
  4. 点击「登录」按钮
- **预期结果**:
  - 返回错误码 `2003`（User Disabled）
  - 提示「账户已被禁用」

### TC-E-007: 验证码输入错误 3 次后需重新获取

- **需求**: 验证码错误 3 次后需重新获取
- **优先级**: Medium
- **前置条件**: 系统可发送验证码
- **测试步骤**:
  1. 输入手机号
  2. 获取验证码
  3. 输入错误验证码 3 次
  4. 再次尝试输入验证码
- **预期结果**:
  - 提示验证码错误次数过多
  - 需要重新获取验证码

### TC-E-008: 验证码过期

- **需求**: 验证码有效期 5 分钟
- **优先级**: High
- **前置条件**: 获取验证码已超过 5 分钟
- **测试步骤**:
  1. 获取验证码
  2. 等待 5 分钟后输入该验证码
  3. 尝试注册/登录
- **预期结果**:
  - 返回错误码 `1004`（Code Expired）
  - 提示「验证码已过期，请重新获取」

### TC-E-009: 手机号格式校验

- **需求**: 手机号必须为 11 位数字
- **优先级**: High
- **前置条件**: 无
- **测试步骤**:
  1. 访问注册页面 `/register`
  2. 输入格式错误的手机号：`1380013800`
  3. 点击「获取验证码」按钮
- **预期结果**:
  - 提示手机号格式不正确
  - 无法发送验证码

### TC-E-010: 未勾选服务协议

- **需求**: 必须同意服务条款才能注册
- **优先级**: High
- **前置条件**: 手机号未注册
- **测试步骤**:
  1. 访问注册页面 `/register`
  2. 输入手机号：`13800138999`
  3. 输入密码：`Abcd1234`
  4. 确认密码：`Abcd1234`
  5. 不勾选「我已阅读并同意服务条款和隐私政策」
  6. 点击「注册」按钮
- **预期结果**:
  - 提示必须同意服务条款
  - 无法提交注册

## 4. Token 测试

### TC-TOKEN-001: Access Token 有效期验证

- **需求**: Access Token 有效期 24 小时
- **优先级**: High
- **前置条件**: 用户已登录，获取到 Access Token
- **测试步骤**:
  1. 登录获取 Access Token
  2. 检查 Token 的有效期或解析 Token
- **预期结果**:
  - Token 有效期为 24 小时
  - 存储在 httpOnly Cookie 中

### TC-TOKEN-002: Refresh Token 有效期验证

- **需求**: Refresh Token 有效期 7 天
- **优先级**: High
- **前置条件**: 用户已登录，获取到 Refresh Token
- **测试步骤**:
  1. 登录获取 Refresh Token
  2. 检查 Refresh Token 的有效期
- **预期结果**:
  - Refresh Token 有效期为 7 天
  - 可用于刷新 Access Token
