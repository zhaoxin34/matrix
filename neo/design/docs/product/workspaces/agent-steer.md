## 🎯 产品概述

### 什么是 Agent Steer

**Agent Steer**（智能体操控台）是 Chrome 扩展中嵌入的 iframe UI，用于与 Content Script 通信、控制 Agent 行为、显示录制状态。

它的名字来源于"驾驶舱（Steer/Cockpit）"的隐喻——用户坐在驾驶座上，操控和观察 Agent 的行为，就像飞行员操控飞机一样。

### 为什么需要 Agent Steer

在 Chrome Extension 架构中，存在多个组件：
- **Popup**：Extension 弹出窗口，用于配置管理
- **Content Script**：注入到目标页面的脚本，负责底层执行
- **Service Worker**：后台服务，负责消息路由

Popup 负责配置（前端地址、后端地址、功能开关），但不应该承担复杂的交互逻辑。而 Content Script 只能执行底层操作，无法直接与用户交互。

因此需要一个 **中间层 UI** 来：
1. 让用户选择 Agent 模式（学习/指导/主动）
2. 控制录制（开始/暂停/停止）
3. 显示实时状态（录制时长、事件计数）
4. 播放回放（rrweb 回放）

这个中间层就是 **Agent Steer**。

## 📖 名词解释

### Agent Steer

> **定义**：嵌入到目标页面的 iframe UI，作为用户与 Content Script 之间的交互桥梁。

**核心职责**：
- 模式选择（学习/指导/主动）
- 录制控制（开始/暂停/停止）
- 状态显示（实时反馈）
- 回放控制（rrweb player）

**不在 Agent Steer 职责范围内**：
- 配置管理（由 Popup 负责）
- 底层执行（由 Content Script 负责）
- 数据持久化（由 IndexedDB 负责）

### 学习模式（Learn Mode）

> **定义**：用户操作被录制并保存，用于后续分析和回放。

**典型场景**：
- 新员工学习业务流程
- 记录专家的操作步骤
- 采集训练数据

### 指导模式（Guide Mode）

> **定义**：根据历史录制数据，在用户操作时提供实时提示和引导。

**典型场景**：
- 辅助用户完成复杂操作
- 提供步骤指引
- 实时纠错

### 主动模式（Active Mode）

> **定义**：Agent 根据预设任务自动执行操作，用户可以随时接管。

**典型场景**：
- 自动完成重复性任务
- 批量处理数据
- 无人值守运行

## 🔧 功能解释

### 1. 模式切换

Agent Steer 提供三种模式的切换入口：

```
┌─────────────────────────────────────┐
│  🔧 Agent Steer                      │
├─────────────────────────────────────┤
│                                     │
│  ┌─────────┐ ┌─────────┐ ┌───────┐ │
│  │  学习   │ │  指导   │ │ 主动  │ │
│  │  Learn  │ │  Guide  │ │Active │ │
│  └─────────┘ └─────────┘ └───────┘ │
│                                     │
└─────────────────────────────────────┘
```

**交互流程**：
1. 用户点击模式按钮
2. Agent Steer 通过 `postMessage` 发送 `SET_MODE` 命令
3. Content Script 接收命令，更新状态
4. Content Script 通过 `postMessage` 同步状态回 Agent Steer

### 2. 录制控制

当处于学习模式时，显示录制控制界面：

```
┌─────────────────────────────────────┐
│  🔧 Agent Steer                      │
├─────────────────────────────────────┤
│  🎬 学习模式                          │
│                                     │
│  ● 00:15:32              ⏸ ⏹      │
│  事件: 128                        │
│                                     │
│  ┌─────────────────────────────┐   │
│  │      录制指示器（Overlay）   │   │
│  │      Recording...          │   │
│  └─────────────────────────────┘   │
│                                     │
└─────────────────────────────────────┘
```

**按钮说明**：
| 按钮 | 状态 | 说明 |
|------|------|------|
| ▶ 开始 | Idle | 点击后开始录制 |
| ⏸ 暂停 | Recording | 点击后暂停录制 |
| ⏹ 停止 | Recording/Paused | 点击后停止录制并保存 |

### 3. 状态显示

Agent Steer 实时显示当前状态：

| 状态 | 显示 | 说明 |
|------|------|------|
| Idle | 灰色圆点 + "空闲" | 未录制 |
| Recording | 绿色脉冲圆点 + "录制中" | 正在录制 |
| Paused | 黄色圆点 + "已暂停" | 录制暂停 |
| Playing | 蓝色圆点 + "回放中" | 正在回放 |

### 4. 回放控制（指导模式/主动模式）

当有录制数据时，可以在指导模式下回放：

```
┌─────────────────────────────────────┐
│  🔧 Agent Steer                      │
├─────────────────────────────────────┤
│  🎥 指导模式                          │
│                                     │
│  ┌─────────────────────────────┐   │
│  │      rrweb Player          │   │
│  │      ▶ 00:05:32 / 00:10:00 │   │
│  └─────────────────────────────┘   │
│                                     │
│  进度条: ════════════●───────────  │
│                                     │
│  操作: 点击了「提交」按钮           │
│                                     │
└─────────────────────────────────────┘
```
