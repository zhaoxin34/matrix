## 🎯 产品概述

### 1.1 组织定义

组织是 Neo 系统的**层级结构**，用于管理和隔离资源。一个组织可以包含多个 Workspace。

### 1.2 组织与 Workspace 的关系

```
┌─────────────────────────────────────────────────────────────────┐
│                         组织 (Organization)                      │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │                                                            │  │
│  │   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    │  │
│  │   │ Workspace A │    │ Workspace B │    │ Workspace C │    │  │
│  │   └─────────────┘    └─────────────┘    └─────────────┘    │  │
│  │                                                            │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                 │
│   成员 (Employee) ←→ User (全局用户池)                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**关系说明**：

- **1:N**：一个组织可以拥有多个 Workspace
- Workspace 必须从属于某个组织
- 组织成员可以访问该组织下的所有 Workspace

### 1.3 组织层级

组织采用**树形结构**，最多支持 4 级：

| 级别 | 类型      | 示例         |
| ---- | --------- | ------------ |
| 1    | 公司/企业 | Matrix 公司  |
| 2    | 部门      | 北京研发中心 |
| 3    | 子部门    | 前端开发组   |
| 4    | 小组      | A 小组       |

### 1.4 核心概念

| 概念                    | 说明                                 |
| ----------------------- | ------------------------------------ |
| **Organization**        | 组织单元，包含公司、部门、小组       |
| **Employee**            | 员工，属于某个组织单元               |
| **User**                | 系统用户，全局身份，与 Employee 解耦 |
| **UserEmployeeMapping** | 用户与员工的关联表                   |

## 👤 组织管理

### 2.1 组织 CRUD

| 功能     | 说明                                 |
| -------- | ------------------------------------ |
| 创建组织 | 在指定父节点下创建子组织             |
| 查看组织 | 查看组织树形结构                     |
| 编辑组织 | 修改组织名称                         |
| 删除组织 | 删除组织（需校验是否有子组织或成员） |
| 移动组织 | 将组织移动到其他父节点下             |

### 2.2 组织属性

| 属性         | 类型     | 必填 | 说明                                   |
| ------------ | -------- | ---- | -------------------------------------- |
| `id`         | UUID     | 是   | 全局唯一标识符                         |
| `name`       | string   | 是   | 组织名称，1-50字符                     |
| `code`       | string   | 是   | 组织编码，唯一，自动生成               |
| `parent_id`  | UUID     | 否   | 父组织 ID，根组织为 null               |
| `level`      | int      | 是   | 层级深度，1-4                          |
| `path`       | string   | 是   | 路径枚举，存储完整路径（如 `/1/2/3/`） |
| `status`     | enum     | 是   | `active` / `disabled`                  |
| `created_at` | datetime | 是   | 创建时间                               |
| `updated_at` | datetime | 是   | 更新时间                               |

### 2.3 组织状态机

```mermaid
stateDiagram-v2
    [*] --> 活跃: 创建组织
    活跃 --> 禁用: 超级管理员禁用
    禁用 --> 活跃: 超级管理员启用
```

| 状态       | 说明                       | 可执行操作             |
| ---------- | -------------------------- | ---------------------- |
| `active`   | 正常状态                   | 查看、编辑、禁用、移动 |
| `disabled` | 已禁用，资源保留但不可操作 | 查看、编辑、启用       |

**约束**：

- 删除组织前需校验是否有子组织或员工
- 禁用组织后，组织下的所有 Workspace 不受影响

## 👤 员工管理

### 3.1 员工属性

| 属性              | 类型     | 必填 | 说明                                    |
| ----------------- | -------- | ---- | --------------------------------------- |
| `id`              | UUID     | 是   | 全局唯一标识符                          |
| `name`            | string   | 是   | 员工姓名                                |
| `employee_no`     | string   | 是   | 工号，唯一                              |
| `primary_unit_id` | UUID     | 是   | 主属组织单元 ID                         |
| `job_title`       | string   | 否   | 岗位名称                                |
| `status`          | enum     | 是   | `onboarding` / `active` / `offboarding` |
| `join_date`       | date     | 否   | 入职日期                                |
| `dimission_date`  | date     | 否   | 离职日期                                |
| `created_at`      | datetime | 是   | 创建时间                                |
| `updated_at`      | datetime | 是   | 更新时间                                |

### 3.2 员工与组织的关系

- **主属部门**：每个员工必须属于一个主属组织单元
- **辅助部门**：员工可以同时属于多个辅助组织单元
- **调动历史**：员工岗位变动时记录历史

### 3.4 员工状态机

```mermaid
stateDiagram-v2
    [*] --> 待入职: 创建员工档案
    待入职 --> 在职: 入职报道
    在职 --> 待离职: 提交离职申请
    待离职 --> 已离职: 办理离职手续
    在职 --> 待入职: 重新入职
```

| 状态          | 说明                 | 可执行操作       |
| ------------- | -------------------- | ---------------- |
| `onboarding`  | 待入职状态           | 编辑、删除       |
| `active`      | 在职状态             | 编辑、调动、离职 |
| `offboarding` | 待离职状态           | 查看、办理离职   |
| `inactive`    | 已离职状态（软删除） | 查看             |

**约束**：

- 员工离职后状态变为 `offboarding`，数据保留
- 已离职员工不可再次入职，需新建档案

## 👤 超级管理员功能

### 4.1 超级管理员的职责

| 功能     | 说明                           |
| -------- | ------------------------------ |
| 组织列表 | 查看所有组织树形结构           |
| 创建组织 | 在指定位置创建新组织           |
| 编辑组织 | 修改组织名称                   |
| 删除组织 | 删除组织（需校验）             |
| 移动组织 | 调整组织层级                   |
| 员工列表 | 查看所有员工                   |
| 员工调动 | 将员工从一个部门调到另一个部门 |
| 批量导入 | Excel 导入员工数据             |
| 批量导出 | Excel 导出员工数据             |
