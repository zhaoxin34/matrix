## 🎯 产品概述

### 1.1 组织定义

组织是 Neo 系统的**层级结构**，用于管理和隔离资源。一个组织可以包含多个 Workspace。

### 1.2 组织与 Workspace 的关系

```
┌─────────────────────────────────────────────────────────────────┐
│                         组织 (Organization)                      │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │                                                            │  │
│  │   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    │  │
│  │   │ Workspace A │    │ Workspace B │    │ Workspace C │    │  │
│  │   └─────────────┘    └─────────────┘    └─────────────┘    │  │
│  │                                                            │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                 │
│   成员 (Employee) ←→ User (全局用户池)                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**关系说明**：

- **1:N**：一个组织可以拥有多个 Workspace
- Workspace 必须从属于某个组织
- 组织成员可以访问该组织下的所有 Workspace

### 1.3 组织层级

组织采用**树形结构**，最多支持 4 级：

| 级别 | 类型      | 示例         |
| ---- | --------- | ------------ |
| 1    | 公司/企业 | Matrix 公司  |
| 2    | 部门      | 北京研发中心 |
| 3    | 子部门    | 前端开发组   |
| 4    | 小组      | A 小组       |

### 1.4 核心概念

| 概念                    | 说明                                 |
| ----------------------- | ------------------------------------ |
| **Organization**        | 组织单元，包含公司、部门、小组       |
| **Employee**            | 员工，属于某个组织单元               |
| **User**                | 系统用户，全局身份，与 Employee 解耦 |
| **UserEmployeeMapping** | 用户与员工的关联表                   |

## 👤 组织管理

### 2.1 组织 CRUD

| 功能     | 说明                                 |
| -------- | ------------------------------------ |
| 创建组织 | 在指定父节点下创建子组织             |
| 查看组织 | 查看组织树形结构                     |
| 编辑组织 | 修改组织名称                         |
| 删除组织 | 删除组织（需校验是否有子组织或成员） |
| 移动组织 | 将组织移动到其他父节点下             |

### 2.2 组织属性

| 属性         | 类型     | 必填 | 说明                                   |
| ------------ | -------- | ---- | -------------------------------------- |
| `id`         | UUID     | 是   | 全局唯一标识符                         |
| `name`       | string   | 是   | 组织名称，1-50字符                     |
| `code`       | string   | 是   | 组织编码，唯一，自动生成               |
| `parent_id`  | UUID     | 否   | 父组织 ID，根组织为 null               |
| `level`      | int      | 是   | 层级深度，1-4                          |
| `path`       | string   | 是   | 路径枚举，存储完整路径（如 `/1/2/3/`） |
| `status`     | enum     | 是   | `active` / `disabled`                  |
| `created_at` | datetime | 是   | 创建时间                               |
| `updated_at` | datetime | 是   | 更新时间                               |

### 2.3 组织层级存储

采用 **Closure Table** 模式存储组织层级关系：

```sql
org_unit_closure
├── ancestor_id: UUID   -- 祖先节点 ID
├── descendant_id: UUID -- 后代节点 ID
├── depth: int          -- 深度（0=自身，1=直接子节点，N=第N代）
```

**查询优势**：

- 获取某组织所有后代：`WHERE ancestor_id = ?`
- 获取某组织所有祖先：`WHERE descendant_id = ?`
- 获取直接子节点：`WHERE ancestor_id = ? AND depth = 1`

## 👤 员工管理

### 3.1 员工属性

| 属性              | 类型     | 必填 | 说明                                    |
| ----------------- | -------- | ---- | --------------------------------------- |
| `id`              | UUID     | 是   | 全局唯一标识符                          |
| `name`            | string   | 是   | 员工姓名                                |
| `employee_no`     | string   | 是   | 工号，唯一                              |
| `primary_unit_id` | UUID     | 是   | 主属组织单元 ID                         |
| `job_title`       | string   | 否   | 岗位名称                                |
| `status`          | enum     | 是   | `onboarding` / `active` / `offboarding` |
| `join_date`       | date     | 否   | 入职日期                                |
| `dimission_date`  | date     | 否   | 离职日期                                |
| `created_at`      | datetime | 是   | 创建时间                                |
| `updated_at`      | datetime | 是   | 更新时间                                |

### 3.2 员工与组织的关系

- **主属部门**：每个员工必须属于一个主属组织单元
- **辅助部门**：员工可以同时属于多个辅助组织单元
- **调动历史**：员工岗位变动时记录历史

### 3.3 员工与用户的关系

```
User (1) ←→ (0..1) UserEmployeeMapping (1) ←→ (1) Employee (N) → Organization
```

- 一个 User 可以绑定一个 Employee
- 一个 Employee 可以绑定一个 User
- Employee 属于某个组织

## 🎨 API 设计

### 6.1 组织 API

| 方法   | 端点                          | 说明       |
| ------ | ----------------------------- | ---------- |
| GET    | `/api/v1/org/units`           | 获取组织树 |
| POST   | `/api/v1/org/units`           | 创建组织   |
| PUT    | `/api/v1/org/units/{id}`      | 编辑组织   |
| DELETE | `/api/v1/org/units/{id}`      | 删除组织   |
| PUT    | `/api/v1/org/units/{id}/move` | 移动组织   |

### 6.2 员工 API

| 方法   | 端点                              | 说明               |
| ------ | --------------------------------- | ------------------ |
| GET    | `/api/v1/employees`               | 获取员工列表       |
| POST   | `/api/v1/employees`               | 创建员工           |
| PUT    | `/api/v1/employees/{id}`          | 编辑员工           |
| DELETE | `/api/v1/employees/{id}`          | 删除员工（软删除） |
| PUT    | `/api/v1/employees/{id}/transfer` | 调动员工           |
| POST   | `/api/v1/employees/import`        | 批量导入           |
| GET    | `/api/v1/employees/export`        | 批量导出           |
