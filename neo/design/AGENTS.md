# 项目介绍

这个项目的目标是管理neo项目的产品文档、技术设计文档、UI文档。技术上采用docusaurus.

**生成日期:** 2026-05-09

## 技术栈

- **框架**: Docusaurus 3.10.1
- **语言**: TypeScript ~6.0.2
- **UI 库**: React 19.0.0
- **内容格式**: MDX (Markdown + JSX)
- **样式框架**: Infima CSS Framework (通过 Docusaurus)
- **语法高亮**: prism-react-renderer 2.3.0
- **Node 版本**: >= 20.0

## 项目结构

```
neo/design/
├── docs/                    # 主要文档 (MDX 文件)
│   ├── product/            # 产品文档
│   ├── ui-design/          # UI设计文档
│   ├── technical/          # 技术文档
│   ├── e2e/                # E2E测试用例
├── src/                     # 源代码
│   ├── pages/              # 自定义页面 (index.tsx, markdown-page.mdx)
│   ├── components/          # 自定义 React 组件 (HomepageFeatures/)
│   └── css/                # 自定义样式 (custom.css)
├── static/                  # 静态资源 (图片, favicon)
│   └── img/                # Logo, 社交卡片, undraw SVG
├── sidebars.ts              # 侧边栏配置
├── docusaurus.config.ts     # Docusaurus 配置
├── tsconfig.json           # TypeScript 配置
├── package.json            # 依赖和脚本
```

## 约定规范

### 文档结构

- 每个分类文件夹都有 `_category_.json` 文件用于元数据
- 侧边栏**自动生成**自目录结构 (`type: 'autogenerated'`)
- MDX 文件支持 React 组件和 JSX 语法
- 必需 frontmatter 用于排序: `sidebar_position: N`

### 文件命名

- MDX 文件: kebab-case (如 `intro.mdx`, `create-a-document.mdx`)
- 分类 JSON: `_category_.json`
- 组件: PascalCase (如 `HomepageFeatures/index.tsx`)

### 样式规范

- 使用 Infima CSS 变量进行主题设置
- 自定义颜色在 `src/css/custom.css` 中定义
- 深色模式通过 `data-theme='dark'` 选择器
- 组件样式使用 CSS Modules (`.module.css`)

### 内容规范

- 网站标题: "Neo 设计文档"
- 默认语言环境: `en` (配置中), 但内容为中文
- 颜色主题尊重系统 `prefers-colorScheme`

### 文档交叉链接规范

产品文档和技术文档之间需要建立双向链接，方便读者跳转查阅。

**链接位置**：每个文档的**最下方**，在 `## 🔗 相关文档` 标题下放置链接。

**链接格式**：使用 Docusaurus 文档路由格式，路径为 `/目录/id`（不含 `.md` 后缀）。

**示例**：

产品设计文档底部：

```markdown
## 🔗 相关文档

- [ Workspace 技术设计 ](../technical/workspace)
```

技术设计文档底部：

```markdown
## 🔗 相关文档

- [ Workspace 产品设计 ](../product/workspace)
- 数据库设计文档 (TODO)
- 权限系统设计文档 (TODO)
```

**命名规范**：

- 产品文档之间的链接使用文档标题（如「Workspace 产品设计」）
- 相关但未创建的文档使用「(TODO)」标注

## 关键文件

| 文件                   | 用途                                                          |
| ---------------------- | ------------------------------------------------------------- |
| `docusaurus.config.ts` | Docusaurus 主配置: 标题、导航栏、页脚、i18n、预设             |
| `sidebars.ts`          | 4 个文档分类的侧边栏结构 (product, ui-design, technical, e2e) |
| `package.json`         | 依赖、脚本、Node 版本要求 (>=20.0)                            |
| `src/css/custom.css`   | 全局 CSS 覆盖和 Infima 颜色变量                               |
| `src/pages/index.tsx`  | 带有 HomepageFeatures 组件的主页                              |
| `docs/intro.mdx`       | 文档介绍页面                                                  |

## 注意事项 (避免事项)

- **不要手动编辑 sidebars.ts** — 侧边栏内容由目录结构自动生成
- **不要删除 `_category_.json` 文件** — 这些定义了分类标签和排序
- **不要添加 ESLint/Prettier 配置** — 项目未使用; Docusaurus 处理代码检查
- **不要更改 `baseUrl: '/'`** — 配置为根路径部署
- **不要使用 CommonJS 语法** — 项目使用 ESM (Docusaurus v3+)
- **避免破坏 MDX 导入** — 组件和导入必须对 MDX 和 React 都有效

## 备注

- **开发服务器运行在端口 3200** (非默认 3000) — `npm run start` 启动于 http://localhost:3200
- **博客功能已启用** — 存在带 authors.yml 和 tags.yml 的博客文章
- **深色模式已启用** — 通过 `colorMode: { respectPrefersColorScheme: true }` 尊重系统偏好
- **v4 兼容性标志已设置** — `future: { v4: true }` 已启用, 为即将到来的 Docusaurus v4 做准备
- **TypeScript 严格模式已启用** — `tsconfig.json` 设置了 `"strict": true`
- **使用更快的 Rspack 打包器** — 包含 `@docusaurus/faster` 包以提升构建性能
