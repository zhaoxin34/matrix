---
description: 检查产品文档的规范是否满足，如果不满足则尝试修改
argument-hint: <doc_path>
---

按照如下步骤检查文档 $1 是否符合规范, 如果存在以下问题，则先要和用户确定哪些问题需要修改，把待修改的问题创建一个TODO list，然后在逐个解决问题

1. use scout subagent: 检查 $1 的文档中的链接是否正确

正确的链接示例`[layout设计](./product/layout)`, 注意没有.md

错误的示例
`[layout设计](./product/layout.md)`

> 注意上面示例中的layout是文档id，示例如下

./product/layout.md 文件内容示例
```
---
id: layout
title: xxx
---
```

2. use scout subagent: 检查 $1 的文档中是否包含了技术设计的内容，比如`API设计`，如果存在，那么移动到`./technical`目录下，如果技术文档已存在，则合并文档，如果合并会导致冲突，则询问一下用户如何解决冲突

3. use scout subagent: 检查 $1 的文档中所描述的实体是否有完整属性设计，比如涉及用户实体，则文档中应该包含完整的用户实体设计，包括`用户`有哪些字段，字段有哪些属性

4. use scout subagent: 检查 $1 的文档中所描述的实体是否有状态的变化，如果有，必须包含状态机设计，状态机设计最好是mermaid图表

5. use scout subagent: 检查 $1 的文档中是否包含了足够的业务假设

6. use scout subagent: 检查 $1 的文档是否存在技术设计文档，如果存在，应该建立相关文档的双向链接

7. use scout subagent: 检查 $1 的文档是否存在技术设计文档，如果存在，是否和当前文档存在冲突或矛盾的设计，比如字段类型不一致

8. use scout subagent: 检查 $1 的文档是否存在页面路由设计，不存在则后续需要创建

路由的设计示例如下，可以参考

```
## xxxx UI 设计

| 页面                | 路由           | 说明           |
| ------------------- | -------------- | -------------- |
| 登录页              | `/login`       | 用户登录       |
```


9. use scout subagent: 检查 $1 的文档是否存已经完成高保真UI原型设计，如果存在，是否已经建立到原型的链接

关联原型的示例如下

```
## 🔗 关联原型

**UI 原型页面**: [组织管理页面](../ui/app/admin/org-structure/page.tsx)

**访问地址**: http://localhost:3300/admin/org-structure
```

> ../ui 是高保真原型的项目目录，它是个react的项目，具体可以去项目下看

如果原型不存在则需要在文档`设计检查清单`标记一下，标记示例如下

```
## ✅ 设计检查清单
- [ ] 设计 UI 原型
```
