## 5. 建议与问题

### 5.1 UI与产品设计不一致问题

| 问题 | 当前UI状态 | 产品设计要求 | 建议 |
|------|------------|--------------|------|
| **员工删除按钮** | 有删除按钮和确认对话框 | 不提供删除功能 | **需确认**：删除功能是否已被加入或设计已变更 |
| **员工调动功能** | UI中无调动入口 | 有调动功能 | 需前端实现或更新测试用例为"暂不支持" |
| **员工离职功能** | UI中表现为删除 | 离职状态变更 | 需明确UI是"删除"还是"离职" |
| **批量导入导出** | 有按钮但无实际功能 | 暂不做 | 用例已正确标注"暂不做" |

### 5.2 测试数据依赖问题

多个测试用例依赖特定的前置数据状态，建议：
- 编写测试数据初始化fixture
- 使用 `@pytest.fixture` 在每个测试前准备所需数据
- 使用数据库备份/恢复机制保证测试独立性

### 5.3 用例执行顺序问题

部分测试用例存在依赖关系（如 TC-E-ORG-001 需要先有3级组织），建议：
- 使用 `pytest-order` 指定执行顺序
- 或将相关用例合并为同一测试方法

### 5.4 元素定位建议

基于前端代码分析，测试用例应使用以下定位策略：

```python
# 组织树节点操作菜单
org_menu = page.locator("button").filter(has=page.locator("svg path[d*='M12 12']"))  # MoreIcon

# 新增按钮
page.get_by_role("button", name="新增").or_(page.get_by_text("新增"))

# 员工操作-编辑
page.locator("button[aria-label='编辑']").or_(page.locator("button").filter(has=page.locator("svg path[d*='17 3a2.85']")))

# 搜索框
page.get_by_placeholder("搜索姓名、工号、手机号...")

# 删除确认
page.get_by_role("button", name="删除")
```
