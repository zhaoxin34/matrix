# 用户-员工映射功能

## Summary

在组织管理模块中，添加用户(User)和员工(Employee)的映射关系。创建员工时必须先选择关联的用户系统账号，并自动同步用户信息（手机号、姓名、邮箱）。

## Motivation

当前系统中，User（用户系统账号）和 Employee（员工档案）是独立的实体，没有建立映射关系。这导致：

1. 无法将组织架构与权限系统关联
2. 员工创建时需重复录入已在用户中存在的信息
3. 同一自然人在系统中存在两条独立记录，造成数据冗余

建立映射关系后，可以实现：
- 员工必须有系统账号，确保组织架构完整性
- 数据一次录入多处复用，减少人工操作
- 权限管理可以基于组织架构进行

## User Story

```
作为 HR 管理员
我希望在创建员工时能够选择已有的系统用户
以便自动填充员工信息并建立关联
```

## Requirements

### 核心需求

1. **用户搜索选择**
   - 支持按手机号搜索用户
   - 已关联的用户显示为禁用状态
   - 未关联的用户可以选中

2. **信息自动同步**
   - 选择用户后，自动填充：姓名、手机号、邮箱
   - 手机号设为只读（强制同步）
   - 姓名、邮箱允许编辑（员工可以有别称）

3. **创建时建立映射**
   - Employee 和 UserEmployeeMapping 在同一事务中创建
   - 确保数据一致性

### 业务规则

| 规则 | 说明 |
|------|------|
| 员工必须关联用户 | 不选择用户，无法创建员工 |
| 映射关系 1:1 | 一个用户只能映射到一个员工 |
| 手机号强制同步 | 员工手机号与用户手机号一致 |
| 已关联用户禁用 | 避免重复关联 |

## Non-Goals

- 修改现有的用户管理功能
- 支持员工离职后自动解绑用户
- 批量关联功能
