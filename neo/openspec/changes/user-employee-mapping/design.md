# 用户-员工映射 - 技术设计

## Overview

本文档详细描述用户-员工映射功能的技术实现方案。

## 数据模型

### 现有模型

```
┌─────────────────┐       ┌───────────────────────────┐       ┌─────────────────┐
│      User        │       │  UserEmployeeMapping      │       │    Employee     │
│─────────────────│       │───────────────────────────│       │─────────────────│
│ id (PK)         │◀──────│ user_id (FK, unique)      │──────▶│ id (PK)         │
│ phone           │       │ employee_id (FK, unique)  │       │ employee_no     │
│ username        │       │ created_at                │       │ name            │
│ email           │       └───────────────────────────┘       │ phone           │
│ ...             │                                          │ email           │
└─────────────────┘                                          │ primary_unit_id │
                                                              │ ...             │
                                                              └─────────────────┘
```

### 现有字段约束

| 模型 | 字段 | 类型 | 说明 |
|------|------|------|------|
| User | phone | String(20), unique | 登录账号 |
| Employee | phone | String(20), nullable | 联系方式 |
| Employee | email | String(100), nullable | 邮箱 |

## API 设计

### 1. 查询未关联用户列表

**需求背景**: 前端需要展示可选的用户列表，已关联的用户需显示但禁用。

**Endpoint**: `GET /api/v1/admin/users/unlinked`

**Query Parameters**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| search | string | 否 | 按手机号/用户名搜索 |
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页数量，默认20 |

**Response**:
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "total": 100,
    "page": 1,
    "page_size": 20,
    "list": [
      {
        "id": 1,
        "phone": "138****8002",
        "username": "张三",
        "email": "zhangsan@example.com",
        "is_active": true,
        "linked_employee": {
          "id": 10,
          "name": "张三",
          "employee_no": "EMP-001"
        }
      }
    ]
  }
}
```

**说明**:
- `linked_employee` 为 null 时表示未关联，可选
- 不为 null 时表示已关联，显示在列表中但禁用

### 2. 获取用户详情

**Endpoint**: `GET /api/v1/admin/users/{user_id}`

**Response**:
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": 1,
    "phone": "13800138002",
    "username": "张三",
    "email": "zhangsan@example.com",
    "is_active": true,
    "linked_employee": {
      "id": 10,
      "name": "张三",
      "employee_no": "EMP-001"
    }
  }
}
```

### 3. 创建员工（扩展）

**扩展现有接口**: `POST /api/v1/employees`

**Request Body**:
```json
{
  "user_id": 1,
  "employee_no": "EMP-2024-001",
  "name": "张三",
  "phone": "13800138002",
  "email": "zhangsan@example.com",
  "position": "工程师",
  "primary_unit_id": 5
}
```

**关键逻辑**:
1. `user_id` 为必填字段
2. 校验用户是否已被其他员工关联
3. `phone` 必须与用户手机号一致（强制同步）
4. 在同一事务中创建 Employee 和 UserEmployeeMapping

**Response**:
```json
{
  "code": 0,
  "message": "员工创建成功",
  "data": {
    "id": 10,
    "employee_no": "EMP-2024-001",
    "name": "张三",
    "phone": "13800138002",
    "email": "zhangsan@example.com",
    "position": "工程师",
    "primary_unit": { "id": 5, "name": "技术部" },
    "user": { "id": 1, "phone": "13800138002" },
    "status": "onboarding",
    "created_at": "2024-01-15T10:00:00Z"
  }
}
```

### 4. 错误码

| 错误码 | 说明 |
|--------|------|
| ERR_USER_ALREADY_LINKED | 用户已被其他员工关联 |
| ERR_USER_NOT_FOUND | 用户不存在 |
| ERR_PHONE_MISMATCH | 手机号与用户不匹配 |

## 前端交互设计

### 添加员工对话框流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        添加员工                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Step 1: 搜索选择用户                                               │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 🔍 搜索: [1380013   ]                                        │   │
│  │                                                             │   │
│  │ ○ 13800138002 - 张三 (已关联 → 员工: EMP-001)                 │   │
│  │ ● 13800138003 - 李四 (可用)                     ← 选中         │   │
│  │ ○ 13800138004 - 王五 (已关联 → 员工: EMP-003)                 │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  Step 2: 自动填充 (选中用户后)                                       │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 工号:    [EMP-2024-001   ]  ← 手动输入                        │   │
│  │ 姓名:    [李四            ]  ← 从用户同步(可编辑)              │   │
│  │ 手机号:  [138****8003    ]  ← 从用户同步(只读) ⚠️             │   │
│  │ 邮箱:    [lisi@xxx.com   ]  ← 从用户同步(可编辑)              │   │
│  │ 职位:    [               ]  ← 手动输入                        │   │
│  │ 部门:    [技术部 ▼        ]  ← 手动选择                        │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│                              [取消]  [确定创建 ✓]                     │
│                              (未选择用户时按钮禁用)                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 交互规则

| 规则 | 说明 |
|------|------|
| 用户必选 | 不选择用户，"确定创建"按钮禁用 |
| 实时搜索 | 输入时实时搜索，300ms 防抖 |
| 列表展示 | 显示用户名、手机号(脱敏)、关联状态 |
| 手机号脱敏 | 显示格式: `138****8003` |
| 自动填充 | 选择用户后立即填充姓名、手机号、邮箱 |
| 只读手机号 | 手机号输入框禁用，不可编辑 |
