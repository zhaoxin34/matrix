## Why

Agent 任务管理是 AI 协作平台的核心功能之一，用于管理 Agent 执行的各种任务（临时任务、周期任务、派发任务）。通过统一的任务管理界面，用户可以查看任务状态、管理任务生命周期、追踪执行记录。

## What Changes

- **任务列表**：支持多维度筛选（状态、类型、优先级、Agent、创建者、时间）和搜索
- **任务详情**：查看任务完整信息、修改周期任务属性、查看执行记录
- **创建周期任务**：通过 Workspace 创建新的周期任务，支持 Cron 表达式配置执行频率
- **编辑周期任务**：修改周期任务的名称、内容、优先级、Cron 表达式（执行中不可编辑）
- **删除周期任务**：删除从未执行过的周期任务（有执行记录的任务只能禁用）
- **状态操作**：取消、禁用、启用任务
- **执行记录**：查看任务的所有执行历史记录

**暂不实现**：
- 暂停/继续功能（UI 按钮存在，点击显示"暂不支持"）
- 周期任务自动执行（需 Agent 模块支持）
- 执行录像回放（需 rrweb 集成）

## Capabilities

### New Capabilities

- `task-management`: 任务管理核心能力，包括任务 CRUD、状态操作、筛选搜索
- `periodic-task`: 周期任务特殊能力，支持 Cron 表达式配置和周期性执行
- `task-execution-record`: 执行记录能力，记录每次任务执行的结果和过程

## Impact

- **后端 API**：13 个 REST API 端点（任务 CRUD、执行记录、状态操作）
- **前端 UI**：4 个页面（任务列表、任务详情、执行记录详情、创建周期任务）
- **数据库**：2 张表（task, task_record）
- **依赖**：
  - 需登录态，但不需要权限控制
  - 周期任务执行依赖 Agent 模块（暂不实现）
  - 暂停/继续依赖 Agent 模块（暂不实现）
