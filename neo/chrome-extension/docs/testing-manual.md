# Chrome Extension Phase 1 手动测试用例

本文档包含手动测试用例 7.5 和 7.6，用于验证 Chrome 扩展的完整录制流程和 iframe 嵌入功能。

## 测试前准备

### 1. 加载扩展到 Chrome

1. 打开 Chrome，访问 `chrome://extensions/`
2. 开启右上角的 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择 `chrome-extension/dist` 目录
5. 确认扩展已加载且无错误

### 2. 启动 Neo 前端

```bash
cd /Volumes/data/working/ai/matrix/neo
cd ui
make dev
```

前端启动后访问 `http://localhost:3300`

## 7.5 手动测试：完整录制流程

### 测试目标

验证 rrweb 录制功能在真实网页上的完整工作流程。

### 测试步骤

#### 步骤 1：检查扩展状态

1. 点击 Chrome 工具栏中的 Neo Agent 图标
2. 确认弹出窗口正常显示
3. 确认状态显示为 "空闲" (Idle)
4. 确认模式选择器显示 "学习模式" / "指导模式" / "主动模式"

#### 步骤 2：开始录制

1. 在 Popup 中选择模式（例如：学习模式）
2. 点击 **开始** 按钮
3. 观察页面左上角是否出现录制指示器（绿色圆点 + "Recording"）
4. 检查录制时长是否开始计时

#### 步骤 3：执行用户操作

在当前页面执行以下操作，观察录制是否正常：

1. **页面滚动** - 向下滚动页面
2. **点击操作** - 点击页面上的任意按钮或链接
3. **输入操作** - 在输入框中输入文本
4. **鼠标悬停** - 将鼠标移动到某个元素上
5. **页面导航** - 如果可以，点击进入子页面

#### 步骤 4：暂停录制

1. 点击录制指示器或通过 Popup 暂停录制
2. 确认指示器变为黄色 "Paused" 状态
3. 继续操作页面，确认无新事件录制

#### 步骤 5：恢复录制

1. 再次点击恢复录制
2. 确认指示器变回绿色 "Recording" 状态
3. 执行一些操作，确认事件被正常录制

#### 步骤 6：停止录制

1. 点击停止按钮或通过 Popup 停止录制
2. 确认指示器消失或变为灰色
3. 确认录制数据已保存到 IndexedDB

#### 步骤 7：验证数据存储

1. 打开 Chrome DevTools (F12)
2. 切换到 Application 标签
3. 选择 IndexedDB > neo-recordings
4. 确认有新的录制记录被创建
5. 检查录制事件数量和内容

### 预期结果

| 检查点 | 预期结果 |
|--------|----------|
| Popup 显示正常 | 界面正确渲染，无 JS 错误 |
| 录制开始 | 指示器出现，状态变为 Recording |
| 操作录制 | 点击、输入、滚动等事件被记录 |
| 暂停功能 | 指示器变为 Paused，新操作不记录 |
| 恢复功能 | 指示器恢复 Recording，新操作被记录 |
| 停止功能 | 指示器消失，录制结束 |
| 数据存储 | 录制数据保存到 IndexedDB |

### 通过标准

- [ ] 所有检查点均通过
- [ ] 无控制台错误
- [ ] 录制数据完整

## 7.6 手动测试：iframe 嵌入流程

### 测试目标

验证 Neo 前端通过 iframe 嵌入到目标网页的功能。

### 测试步骤

#### 步骤 1：检查 Popup 状态

1. 点击 Neo Agent 图标打开 Popup
2. 确认当前状态为空闲
3. 记录当前 URL（稍后用于测试）

#### 步骤 2：启动 iframe

1. 在 Popup 中选择一个模式
2. 点击 **开始** 或 **启动** 按钮
3. 等待 iframe 加载完成

#### 步骤 3：验证 iframe 创建

1. 打开 DevTools Elements 面板
2. 检查页面底部是否创建了新的 `<iframe>` 元素
3. 确认 iframe 具有以下属性：
   - `src` 包含 Neo 前端 URL (localhost:3300)
   - `src` 包含正确的模式参数 (?mode=xxx)
   - `style` 包含固定定位样式（底部、宽度 100%）

#### 步骤 4：验证 iframe 内容

1. 等待 iframe 加载完成（观察 loading 状态消失）
2. 点击 iframe 使其获得焦点
3. 确认 Neo 前端界面正确显示
4. 如果有认证，验证登录界面或主界面

#### 步骤 5：测试 iframe 通信

1. 在 Neo 前端界面执行一些操作
2. 观察主页面是否有响应
3. 检查控制台是否有 postMessage 通信日志

#### 步骤 6：测试关闭功能

1. 在 iframe 中找到并点击 **关闭** 按钮
2. 确认 iframe 被销毁
3. 确认页面恢复正常布局

#### 步骤 7：测试 ESC 快捷键

1. 重新启动 iframe
2. 按下 ESC 键
3. 确认 iframe 被关闭

### 预期结果

| 检查点 | 预期结果 |
|--------|----------|
| iframe 创建 | 底部出现 iframe，src 正确 |
| 前端加载 | Neo 前端界面正常显示 |
| 通信工作 | postMessage 消息正常传递 |
| 关闭按钮 | 点击后 iframe 被移除 |
| ESC 快捷键 | 按 ESC 关闭 iframe |
| 状态同步 | Popup 状态与 iframe 状态一致 |

### 通过标准

- [ ] iframe 正确创建和销毁
- [ ] Neo 前端正常加载
- [ ] 通信机制正常工作
- [ ] 关闭功能正常

## 故障排查

### 扩展加载失败

**错误**: `Could not load file 'content.js' for content script. It isn't UTF-8 encoded.`

**解决方案**:
1. 删除 dist 目录：`rm -rf dist`
2. 重新构建：`make build`
3. 重新加载扩展

### iframe 无法加载

**检查项**:
1. Neo 前端是否运行在 localhost:3300
2. 网络请求是否被阻止
3. CORS 配置是否正确

### 录制事件丢失

**检查项**:
1. 检查 console 日志
2. 确认 storage 模块是否正常工作
3. 检查 IndexedDB 存储配额
