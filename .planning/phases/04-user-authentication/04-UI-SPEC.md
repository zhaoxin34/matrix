## Pages & Components

### 1. Login Page (`/login`)

**Layout:** Centered card, max-width 400px, padding 48px 24px

| Element | Specification |
|---------|---------------|
| Card | `bordered`, shadow on hover, no border-radius override |
| Title | `<Title level={2}>登录</Title>`, centered |
| Subtitle | `<Paragraph>欢迎回来</Paragraph>`, color #666 |
| Phone/Email Input | `size="large"`, prefix icon `UserOutlined`, placeholder "手机号或邮箱" |
| Password Input | `size="large"`, prefix icon `LockOutlined`, `Input.Password`, placeholder "密码" |
| Submit Button | `type="primary"`, `size="large"`, `block`, text "登录" |
| Loading State | `loading` prop during submission |
| Error Message | Ant Design `message.error()` with server response |
| Footer Link | "还没有账号? <Link to="/register">立即注册</Link>" |

**Form Validation:**
- Phone/Email: required, message "请输入手机号或邮箱"
- Password: required, message "请输入密码"

**Interactions:**
- Enter key submits form
- Button shows spinner during API call
- Error message appears at top on failure ( AUTH-03: failed login tracking)

### 2. Register Page (`/register`)

**Layout:** Centered card, max-width 400px, padding 48px 24px

| Element | Specification |
|---------|---------------|
| Card | Same styling as Login |
| Title | `<Title level={2}>注册</Title>`, centered |
| Subtitle | `<Paragraph>创建新账号</Paragraph>`, color #666 |
| Name Input | `size="large"`, prefix `UserOutlined`, placeholder "用户名" |
| Email Input | `size="large"`, prefix `MailOutlined`, placeholder "邮箱" |
| Password Input | `size="large"`, prefix `LockOutlined`, `Input.Password`, placeholder "密码" |
| Phone Input | `size="large"`, prefix `PhoneOutlined`, placeholder "手机号", optional |
| SMS Code Input | Appears after phone filled, `size="large"`, 6-digit, prefix `SafetyOutlined` |
| SMS Send Button | Adjacent to code input, text "发送验证码", countdown timer after sent |
| Terms Checkbox | `<Checkbox>我已阅读并同意<Link>《服务条款》</Link></Checkbox>` |
| Submit Button | `type="primary"`, `size="large"`, `block`, text "注册" |

**Form Validation (AUTH-01):**
- Name: required, message "请输入用户名"
- Email: required + email format, message "请输入有效的邮箱地址"
- Password: required + min 8 chars, complexity (letter + number), message "密码至少8位，需包含字母和数字"
- Phone: optional, format validation (Chinese mobile: 1XX-XXXX-XXXX)
- SMS Code: required when phone provided, 6 digits
- Terms: required checked, message "请同意服务条款"

**SMS Verification (AUTH-02):**
- Rate limit: 5 requests per phone per hour
- Code: 6-digit numeric
- Expiration: 5 minutes
- UI shows countdown timer after sent

### 3. Password Reset Flow

#### 3a. Forgot Password Page (`/forgot-password`)

**Layout:** Centered card, max-width 400px

| Element | Specification |
|---------|---------------|
| Title | `<Title level={2}>找回密码</Title>` |
| Phone Input | `size="large"`, placeholder "请输入手机号" |
| SMS Code Input | 6-digit with send button |
| New Password | `Input.Password`, min 8 chars, complexity required |
| Confirm Password | `Input.Password`, must match new password |
| Submit Button | `type="primary"`, `block`, text "重置密码" |

**Rate Limit (AUTH-06):** 3 requests per phone per day

#### 3b. Reset Success Page

| Element | Copy |
|---------|------|
| Heading | "密码重置成功" |
| Body | "您的密码已成功修改，请使用新密码登录" |
| CTA Button | "返回登录页" (links to /login) |

### 4. Logout

**Interaction:** No dedicated page. Header component shows user name + "退出登录" link when authenticated.

| Element | Specification |
|---------|---------------|
| User Menu | Dropdown with username display |
| Logout Item | Click triggers token invalidation, redirects to /login |

## Error States

| Error | Copy | Solution Path |
|-------|------|---------------|
| Network Error | "网络连接失败，请检查网络" | Retry button |
| Invalid Credentials | "手机号或密码错误" | Link to forgot password |
| Account Locked (AUTH-03) | "登录失败次数过多，账号已锁定15分钟" | Countdown + link to forgot password |
| Rate Limited | "操作过于频繁，请稍后再试" | Countdown timer |
| SMS Limit Reached | "验证码发送次数超限，请明天再试" | Tomorrow link |
| Token Expired | "登录已过期，请重新登录" | Redirect to login |
| Invalid SMS Code | "验证码错误或已过期" | Resend link |

## Rate Limit UI Indicators

When rate limited (AUTH-08):

| Endpoint | Limit | UI Feedback |
|----------|-------|-------------|
| Login | 5/IP/15min | Show attempt count, lockout countdown after 5th fail |
| Register | 3/IP/hour | Show attempt count |
| Password Reset | 3/phone/day | Show remaining attempts |

## Auth Header Integration

After successful login (AUTH-03, AUTH-04):

**Header shows:**
- Logo/Home link (left)
- Search bar (center, if implemented in Phase 5)
- User dropdown (right): avatar/initials + username + caret
  - Dropdown items: "我的账户", "我的订单", divider, "退出登录"

**Logout flow (AUTH-05):**
1. Click "退出登录"
2. Clear access token from storage
3. Clear refresh token (httpOnly cookie or storage)
4. Redirect to /login
5. Show success message "已退出登录"
