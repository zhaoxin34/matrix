## Interaction Contracts

### Checkout Flow

1. User on `/checkout` sees cart items (Step 0 of Steps component)
2. Items listed with name, quantity, unit price, subtotal
3. Total displayed prominently in red (#ff4d4f)
4. Address form below items
5. Submit button: "提交订单" — primary, block, large
6. On submit: Button enters loading state, form disabled
7. On API success: message.success('订单提交成功'), cart cleared, navigate('/orders')
8. On API error: message.error('订单提交失败'), button re-enabled

**Note:** Steps component shows 3 steps but payment step is skipped (auto-paid per D-03). After submit, Steps updates to show "完成" step.

### Address Selection (future enhancement, not v1.2)

- Dropdown of saved addresses + "Add new address" option
- Selected address pre-fills the form
- New address form is the same fields as current inline form
- For v1.2: inline address form only (no saved address selection)

### Order Status Updates

- Admin side: separate admin phase handles shipped/delivered transitions
- User side: read-only status display with colored Tag

### Empty States

- **Checkout (empty cart):** centered text "购物车是空的" — no Empty component, just Paragraph
- **Order List (no orders):** Ant Design `<Empty description="暂无订单">` + `<Paragraph style={{ color: '#666' }}>快去购物吧</Paragraph>` + "去购物" Button linking to /products

### Order Row Click

- Clicking an order row navigates to `/orders/:id`
- No modal/detail drawer — full page navigation
